# Post-Test Review Notes — Phase 1 and Phase 2

This file captures the most useful follow-up outcomes from manual verification of:

- `R25 — Phase 1 multi-agent workspace foundation`
- `R26 — Phase 2 workflow inbox and approval layer`

These notes are intentionally lighter than product planning files. Keep durable strategic follow-up in project product files or ignored private notes.

---

## Confirmed Follow-Up Outcomes

### 1. Agent and mode selection needed clearer structure

Outcome:
- The earlier dropdown-heavy selection flow was too opaque.
- The control panel now uses a more explicit agent-first and mode-aware layout.

What remains useful:
- Keep agent selection understandable before thread entry.
- Keep mode choice visibly tied to the selected agent.
- Preserve honest distinctions between live and deterministic role surfaces.

---

### 2. Routed findings needed to stay visible as workflow state

Outcome:
- Experience Designer findings and related handoff artifacts should remain visible through shared workflow surfaces rather than disappearing into hidden state.

What remains useful:
- Routed findings should stay visible enough that the Product Director can understand why work is still active.
- Additional review surfaces should only be added if Inbox, Orchestrator, and project workflow views stop being sufficient.

---

### 3. Automatic PM clarifications were worth keeping, but should stay reviewable in practice

Outcome:
- PM clarification generation moved toward a more automatic workflow when ambiguity is materially blocking.

What remains useful:
- Check periodically that the threshold for escalating a durable clarification still feels right.
- Keep the Inbox clarification flow understandable and lightweight.

---

### 4. Manual verification needed to become an in-product workflow

Outcome:
- This follow-up turned out to be high value.
- Manual verification now lives inside the control panel rather than depending entirely on external markdown flow.

What remains useful:
- Keep requirement-level verification evidence clear and editable.
- Keep guided verification lightweight and contextual rather than turning it into a second testing system.

---

## Retired or Moved Follow-Ups

These earlier post-test ideas are no longer active manual follow-up items here:

- explicit mode-inference exploration was deferred from the immediate product path
- deeper roadmap and sequencing ideas moved into private planning rather than this manual review file

## How To Use This File

Use this note as a compact historical record of what Phase 1 and Phase 2 testing taught us.

Do **not** use it as the main place for:

- active roadmap planning
- V2 strategy notes
- detailed future ideation
- private product-direction reasoning

Those belong in product files or ignored `private/` notes instead.
