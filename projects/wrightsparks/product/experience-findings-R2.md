# Experience Designer Findings — R2

Mode: Pre-implementation usability review  
Status: accepted  
Requirement: R2

## User problem

Homeowners and businesses scanning the first screen need immediate confirmation that Wright Sparks is local to Reading and Berkshire, carries out dependable electrical work, and communicates professionally. The current hero is attractive but leads with a creative lighting message rather than the requirement's stronger trust and reliability promise.

## Affected user and workflow

The change affects the first step of the existing journey: arriving on the home page, deciding whether the business is relevant and credible, then choosing to request a quote or call. No navigation or interaction change is needed.

## Evidence

- R2 supplies approved, exact hero messaging and identifies Reading and Berkshire as the target area.
- The current hero already supports an eyebrow, two-line headline treatment, supporting paragraph, and contact actions.
- Existing trust evidence and contact routes remain immediately below or within the hero.

## Assessment

- Confidence: High
- Severity / frequency: High-frequency first-impression issue
- Finding type: Clutter / hierarchy and content clarity issue
- Recommendation type: UX improvement in scope

## Recommendation

Replace the hero eyebrow, headline, second line, and supporting copy with the exact R2 wording. Preserve the current CTA order, proof line, imagery, responsive stacking, and semantic heading structure so the content update does not alter functionality.

## Rationale and routing

The approved copy gives the visitor local fit, quality expectation, breadth of work, and working values in the first viewport. This is a bounded content improvement, so route it to UI Designer for typography and responsive constraints, then PM for tasking.
