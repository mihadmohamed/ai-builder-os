# React Best Practices

Use this capability when the work depends on reusable component structure and predictable interaction state.

Focus:

- clear component boundaries
- prop and state clarity
- avoiding duplicated view logic
- keeping app-shell and leaf-component concerns separated

Good default expectations:

- keep components small enough to reason about
- avoid mixing unrelated layout, state, and data responsibilities
- favor reusable patterns over one-off screen code
- preserve readability for the next engineer
