# Frontend App Builder

Use this capability when the work is primarily about shaping or implementing a browser-native product surface.

Focus:

- app-shell structure
- page and route composition
- reusable UI sections
- forms, actions, and feedback states
- responsive layout behavior

Good default expectations:

- build with a Vercel-compatible Next.js/React shape
- keep component boundaries understandable
- treat loading, empty, success, and error states as real UI work
- prefer compositional reuse over page-local duplication
