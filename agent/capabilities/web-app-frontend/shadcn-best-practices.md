# shadcn/ui Best Practices

Use this capability when the work benefits from a component-system approach rather than custom UI from scratch.

Focus:

- consistent component composition
- predictable spacing and hierarchy
- button, form, and feedback consistency
- restrained theme usage

Good default expectations:

- treat components as composable building blocks
- prefer consistency over decorative variation
- keep styling decisions in service of hierarchy and usability
- avoid adopting component patterns that do not fit the real workflow
